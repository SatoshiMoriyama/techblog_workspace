# Anti AI Smell（AI臭さ排除ガイド）

AI生成文章に頻出するパターンを検出し、人間らしい文章に修正するためのチェックリスト。

参考: https://qiita.com/minorun365/items/699e89544da8b0de300d

## 1. 禁止パターン（文体）

| 禁止パターン | なぜダメか | 代替 |
|-------------|-----------|------|
| 太字（`**...**`）の多用 | AIが「重要」と判断した箇所を機械的に太字にしがち | 本当に強調が必要な箇所のみ。基本はベタ書き |
| ダッシュ（—、──、―、-での区切り） | AI特有の区切り表現 | 「…」、読点、または文を分ける |
| 「項目名: 説明」のセット形式 | AIのリスト化癖の典型 | 本文ベタ書き、または見出しレベルで構造化 |
| タイトル・見出しでのコロン多用 | AIが構造化しようとして多用する | カギカッコ、読点で繋ぐ |
| 見出し・タイトルの「です・ます」調 | 全部敬体だと冗長でAIっぽい | 言い切り・体言止め（「〜できる」「〜が必要」） |
| 体言止めの連続 | 名詞止めが連続すると標語の貼り紙化 | 動詞で言い切る、本音を混ぜ、文末トーンを散らす |
| 「！」の多用 | AIが親しみやすさを演出しようとする | 自然な文末表現。本当に強調したい時だけ |
| 過剰な接続詞の連打 | 「さらに」「加えて」「また」「一方で」が毎段落 | 文の流れで自然につなぐ。不要な接続詞は削る |
| 「〜することができます」 | AI頻出の冗長表現 | 「〜できます」で十分 |
| 「〜を活用する」の乱用 | AIが「使う」の代わりに多用 | 素直に「使う」「利用する」 |
| 「〜についてご紹介します」 | AI文章の定型オープニング | いきなり本題に入る |

## 2. 禁止パターン（構成）

- アジェンダ・目次の自動挿入: 短い文章では不要
- まとめセクションで本文の繰り返し: 読めば分かることを繰り返さない
- 均等に網羅的なリスト: 重要なものだけ厳選する
- 「以下の通りです」→ 箇条書き: 毎回このパターンだと機械的
- 結論の先出し + 箇条書き展開: AI文章の定型構造。文脈に応じて構成を変える

## 3. 人間味を出すテクニック

### 個人の経験・考察

- 「実際に触ってみた」姿勢を貫く。評論ではなく手で動かしてから書く
- うまくいかなかった点も正直に書く（失敗談は最高の人間味）
- 「個人的に注目しているのは〜」「これは正直微妙で〜」など主観を恐れない
- 背景やストーリーを添える（なぜこれを書こうと思ったか）

### 比喩と噛み砕き

- 専門用語は初出時に日本語補足をカッコ書きで添える
- 「すごく雑に言うと」方式で厳密な定義をかみ砕く
- 比喩は身近で大胆に使う
- 抽象 → 具体 → 比喩の3段階で説明

### 読者に寄り添う

- 安心フレーズ:「他はデフォルトのまま」「ここだけ押さえればOK」
- 先回りトラブルシューティング: ハマりそうなポイントを事前に説明
- 前提知識を仮定しすぎない

## 4. 情報密度のコントロール

- 箇条書きは3〜4項目が上限。それ以上は分割か本文で説明
- 階層は1段まで。深くても2階層
- 各項目は1〜2文。長い項目は本文に戻す
- 1段落1トピック。詰め込みすぎない

## 5. 表記ルール

- インラインコード（`` ` ``）の前後に半角スペースを入れる
- コードブロックには必ず言語を指定する
- 曖昧な動作主は明示する（NG:「設定される」→ OK:「ユーザーが設定する」）
- 受動態より能動態を優先
